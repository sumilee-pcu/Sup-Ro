## Context

기존 `TripPlan.places`는 후보와 채택 장소를 구분하지 않고 모든 장소를 일정에 넣는다. `selectedPlaceId`도 지도·카드 포커스일 뿐 채택 상태가 아니다. 공개 데모는 외부 키 없이 재현해야 하므로 이번 변경의 추천 결과는 언어모델이 아니라 버전 고정 fixture와 결정적 점수 함수로 만든다.

원당중학교 각종양식 게시판의 최신 현장체험학습 첨부는 2025년 `HWPX`이며 개인 학생이 제출하는 학교장허가 교외체험학습 신청서·보고서다. 수업로 AI는 학생·보호자 개인정보를 수집하지 않으므로 자동 매핑 범위를 날짜, 목적지, 목적, 학습형태와 학습계획으로 제한해야 한다.

## Goals / Non-Goals

**Goals:**

- 하드 제약을 점수와 분리해 차단·검토·선택 가능 상태를 설명한다.
- 같은 입력은 같은 순위와 이유를 만들고 동률 순서도 안정적으로 유지한다.
- 후보 선택이 계획 버전, 일정, 경로, 비용, 승인·문서 상태에 일관되게 반영된다.
- 학교 양식에 옮길 수 있는 비개인정보 초안을 A4 인쇄/PDF로 제공한다.

**Non-Goals:**

- 실제 Kakao 장소검색, 경로 API 또는 언어모델 호출을 이번 fixture 고도화에 연결하지 않는다.
- 학교 원본 HWPX를 저장소에 복제하거나 HWPX 바이너리를 생성하지 않는다.
- 이름, 연락처, 관계, 동의, 서명 또는 체험 후 소감을 추정·자동 작성하지 않는다.

## Decisions

### 1. 후보 풀과 채택 일정을 분리한다

`candidatePlaces`는 출발지를 제외한 탐색 결과를 보존하고, `selectedPlaceIds`는 교사가 일정에 채택한 후보를 나타낸다. 기존 `places`는 출발지와 채택 장소만 포함해 일정 엔진과 기존 문서 렌더러의 의미를 유지한다. `selectedPlaceId`는 하위 호환을 위해 화면 포커스로 남긴다.

### 2. 추천은 결정적 다차원 점수와 하드 게이트를 사용한다

교육과정 연계, 이동 부담, 비용, 접근성, 실내 대안, 운영시간·근거 상태를 0~100 점수와 축별 점수로 기록한다. 필수 접근성 불충족과 제외 후보는 점수에 관계없이 `blocked`, 미확인 핵심 정보는 `review`, 나머지는 `eligible`로 분류한다. 순위는 eligibility, 총점, 장소 ID 순으로 안정화한다.

### 3. 선택은 새 불변 계획 버전을 만든다

선택 액션은 필수 장소와 차단 후보를 검증하고 출발지+선택 장소로 경로·일정·입장료를 다시 계산한다. 버전을 증가시키고 현재 승인과 문서 산출물을 무효화한다. 후보 포커스 변경은 계획 내용 변경이 아니므로 버전을 만들지 않는다.

### 4. 학교 출력은 별도 초안 문서 타입으로 제공한다

`school-application-draft`는 계획이 승인돼도 항상 초안이다. 공식 2025 양식의 날짜, 목적지, 학습형태, 목적, 학습계획만 채우고 개인정보·확인·동의·서명은 빈칸으로 명시한다. 이 문서가 개인 교외체험학습용이며 단체 현장수업 운영계획서의 대체 여부는 학교 확인이 필요하다는 경고와 원본 출처를 포함한다.

### 5. 인쇄용 HTML은 의미 구조를 가진다

Markdown을 단순 이스케이프한 `<pre>`로 넣지 않고 제목, 안내문, 목록, 표, 체크 항목으로 변환한다. 모든 사용자 유래 문자열은 HTML 이스케이프하고 A4 여백, 표 머리글, 페이지 나눔, 초안 워터마크를 적용한다.

## Risks / Trade-offs

- fixture 좌표 기반 이동 점수는 실제 도로 시간과 다르다. → `FIXTURE v1`, 추정, 공식 재확인 상태를 계속 표시한다.
- 후보 선택 후 새 조합의 경로는 직선거리 추정이다. → 경로 근거를 추정 상태로 유지하고 승인 전 확인 대상으로 남긴다.
- 학교별·연도별 양식이 바뀐다. → 문서에 학교, 연도, 원본 URL, 조회 기준일을 고정 표시한다.
- HWPX를 직접 제공하지 않는다. → 이번 범위는 복사 가능한 인쇄/PDF 작성용 초안이며 한컴 원본과의 직접 병합은 별도 검증 과제로 남긴다.
