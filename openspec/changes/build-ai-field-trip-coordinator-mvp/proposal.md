## Why

교사가 체험학습을 설계하려면 교육과정 성취기준, 장소의 교육적 가치, 이동시간, 예산, 안전, 접근성, 날씨 대안을 여러 서비스에서 수작업으로 확인해야 한다. 다음 달 임용 이후 강의·연구·산학협력에서 반복 활용할 수 있도록, 수업로 AI의 첫 서비스로 대한민국 공간정보와 교육 데이터를 근거로 계획을 만들고 모든 추천의 출처와 제약조건 충족 여부를 보여주는 지도 기반 에이전트 MVP가 필요하다.

## What Changes

- 학년, 교과, 성취기준, 출발지, 일정, 인원, 예산, 교통수단, 식사, 안전·접근성 요구를 구조화해 입력하는 계획 작업공간을 제공한다.
- 교육과정 근거와 장소별 학습활동을 연결해 체험학습의 교육적 타당성을 설명한다.
- 국내 장소·관광·공공시설·경로 데이터를 공급자 어댑터로 조회하고 지도와 일정표에 동기화한다.
- 이동시간, 운영시간, 예산, 체류시간, 이동 순서를 함께 검증해 실행 가능한 일정과 대안을 생성한다.
- 기상·대기질·대피시설·접근성 정보를 근거와 함께 표시하고 우천·교통 장애 대안을 제시한다.
- 에이전트의 도구 호출, 데이터 출처, 조회 시각, 가정, 누락 정보를 사용자에게 공개하고 교사의 승인 전에는 최종 계획으로 확정하지 않는다.
- 완성 계획을 교사용 운영계획서, 학생 활동지, 보호자 안내문 초안과 공유 가능한 Markdown/PDF로 내보낸다.
- 학생 개인의 실시간 위치나 민감정보를 수집하지 않는 데모 안전 기본값과 운영·평가 로그를 적용한다.

## Capabilities

### New Capabilities

- `planning-workspace`: 체험학습 제약조건 입력, 계획 초안 생성, 수정, 대안 비교, 저장 상태를 관리한다.
- `curriculum-grounding`: 교육과정 성취기준과 장소·활동을 근거 기반으로 연결하고 학습목표와 활동지를 생성한다.
- `place-route-discovery`: 대한민국 장소, 관광자원, 공공시설, 경로 정보를 공급자 API에서 조회하고 지도에 표시한다.
- `itinerary-constraint-engine`: 시간, 운영시간, 이동, 예산, 인원, 체류시간 제약을 검증하고 실행 가능한 일정을 산출한다.
- `safety-accessibility-resilience`: 안전·기상·대기질·대피·접근성 요구를 점검하고 경고와 대체 계획을 제공한다.
- `agent-provenance-control`: 에이전트 도구 실행, 출처, 가정, 신뢰도, 실패 상태와 사람 승인 절차를 관리한다.
- `plan-documents-export`: 지도·일정·예산을 포함한 교사용 운영계획서, 학생 활동지, 보호자 안내문 초안을 미리보기하고 내보낸다.
- `privacy-evaluation-operations`: 최소정보 처리, 비밀정보 보호, 감사로그, 품질평가, 크로스플랫폼 데모 운영 기준을 적용한다.

### Modified Capabilities

- 없음. 신규 프로젝트의 최초 기능 명세다.

## Impact

- 신규 반응형 웹/PWA, API 서버, 공간 데이터 저장소, 에이전트 도구 계층과 문서 생성 계층이 추가된다.
- 지도·로컬·경로 공급자, 관광·학교·기상·환경·안전 공공데이터 공급자와의 연동이 필요하다.
- 지도 및 공공데이터 공급자는 교체 가능한 어댑터로 분리하고 API 키는 서버 전용 비밀정보로 관리한다.
- 위치정보·공공데이터·지도 타일의 이용정책과 출처 표시를 준수해야 하며, 공개 서비스 전에는 최신 위치정보 관련 의무를 별도 검토한다.
- MVP는 자동 예약·결제, 학생 실시간 위치 추적, 학생 명단·건강정보 수집, 학교 결재 시스템 연동을 포함하지 않는다.
