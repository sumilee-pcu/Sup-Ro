# 3분 시연 스크립트

## 0:00–0:30 문제와 입력

`pnpm dev`로 시작한 뒤 상단의 `FIXTURE v1`을 먼저 설명한다. 중학교 2학년 과학, 생태계 학습 목표, 출발·귀교 시각, 1인 예산, 무단차 동선 조건을 보여준다.

## 0:30–1:20 지도와 일정

중앙 지도 마커를 선택해 왼쪽 일정 카드와 오른쪽 장소 설명이 같은 내부 ID로 동기화됨을 보여준다. 장소 잠금·순서 변경·제외가 계획 버전을 올리고 영향을 받는 구간을 재검증한다는 점을 설명한다.

## 1:20–2:10 근거와 에이전트

오른쪽에서 `fixture/confirmed/estimated/unverified`를 구분하고 조회 시각을 확인한다. 교육과정, 장소, 경로, 기상, 대기질, 안전시설 도구가 허용 목록 안에서 실행되며 모델이 승인·예약·결제를 할 수 없음을 설명한다.

## 2:10–2:40 사람 승인

차단 finding이 없을 때만 `교사 승인` 버튼이 활성화됨을 보여준다. `OFFICIAL_RECHECK`는 출발 전 재확인 체크리스트로 남고 안전 보장 문구로 바뀌지 않는다.

## 2:40–3:00 문서

운영계획서, 학생 활동지, 보호자 안내문을 전환한다. 승인 전 `검토용 초안`, 승인 후 `승인본`이 되며 Markdown 저장과 인쇄/PDF 경로를 보여준다.

## 실패 시연 선택지

- 1인 예산을 1,000원으로 바꿔 `BUDGET_LIMIT` 차단을 표시한다.
- fixture의 운영시간 또는 접근성 상태를 테스트로 변경해 승인 차단을 보여준다.
- 라이브 공급자 실패 테스트가 오류를 숨기지 않고 fixture 모드를 명시하는 것을 `pnpm test` 결과로 보여준다.
