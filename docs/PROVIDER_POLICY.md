# 지도·공공데이터 공급자 정책 메모

확인 기준일은 2026-07-13이다. 이 문서는 법률 자문이나 공급자 계약을 대체하지 않으며 라이브 활성화 직전에 각 공식 문서에서 다시 확인한다.

## Kakao Maps·Local·경로

- 지도 JavaScript 키만 `NEXT_PUBLIC_KAKAO_MAP_APP_KEY`로 브라우저에 전달한다.
- Local·Mobility REST 키는 서버 전용이며 응답·로그·문서에 포함하지 않는다.
- 지도, 장소, 경로 공급자 ID를 내부 장소 ID와 분리한다.
- [Kakao Maps API 공통 문서](https://developers.kakao.com/docs/ko/kakaomap/common)의 도메인 등록, 표시 의무, 쿼터, 캐시 제한을 활성화 시 확인한다.
- Kakao가 공지한 [2026-07-21 신규 지도 API·무료 쿼터 정책](https://devtalk.kakao.com/t/api-notice-on-new-kakao-map-api-features-and-free-quota-policy/150222)에 따라 대중교통·도보·자전거·정적 지도와 앱별 무료 범위가 달라질 수 있다. 앱 활성화 순서, BizWallet, 초과 요금, 일일 상한은 키 발급 시 다시 기록한다.
- 실제 전세버스 운행 가능성, 승하차 위치, 통행 제한은 지도 경로만으로 확정하지 않는다.

## 기상·대기질·안전시설

- 공공데이터 포털 또는 기관 키는 서버 전용으로 보관한다.
- 측정값과 예보값을 구분하며 조회 시각과 예보 유효 범위를 기록한다.
- 자동 조회는 안전 보장이 아니다. 학교 절차, 기상특보, 장소·운송기관 공식 안내를 함께 확인한다.
- 응급·안전시설 위치는 참고 정보로만 표시하고 현장 대응 기관을 자동 지정하지 않는다.

## 캐시·장애 처리

1. 공급자 정책이 재사용을 허용한 경우에만 캐시를 읽는다.
2. 만료 캐시는 최신 사실처럼 표시하지 않고 `cache`와 만료 경고를 함께 반환한다.
3. 라이브 실패 시 호환 캐시가 없으면 명시적으로 버전 고정된 `fixture`만 사용한다.
4. fixture 전환은 실패 원인, 데이터 모드, 사용 불가능한 정보를 사용자에게 표시한다.
5. 테스트는 무제한 실 API 호출 대신 정책상 허용되는 최소 기록 응답과 합성 fixture를 사용한다.

## 활성화 체크리스트

- [ ] Kakao 개발자 앱과 웹 도메인 등록
- [ ] 활성화 앱 순서·무료 쿼터·초과 단가·월 비용 상한 기록
- [ ] 지도 귀속 표시와 캐시·파생 데이터 제한 확인
- [ ] 공공데이터 활용신청과 키 발급
- [ ] API별 호출 제한·예보 범위·장애 응답 기록
- [ ] 서버 로그의 키·전체 응답·사용자 위치 최소화 확인
- [ ] fixture 롤백을 실제로 실행하고 상태 표시 확인
