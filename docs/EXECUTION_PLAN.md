# 수업로 AI 무승인 실행 계획

## 실행 가능성 판단

fixture-first MVP는 순차적으로 완성할 수 있다. 다수의 자율 에이전트를 병렬로 운영할 필요는 없으며, 제품 내부에서는 하나의 결정적 오케스트레이터가 교육과정, 장소, 경로, 일정, 안전, 문서 도구를 제한된 순서로 호출한다. 이렇게 해야 결과 재현, 실패 추적, 비용 통제, 사람 승인 게이트를 유지할 수 있다.

## 단계별 프로세스

| 단계                | 목표                               | 무승인 완료 조건                             |
| ------------------- | ---------------------------------- | -------------------------------------------- |
| P0 기준선           | 네이밍·범위·게이트 확정            | 작업 브랜드, PRD, OpenSpec, 게이트 표 완료   |
| P1 기반             | 크로스플랫폼 앱·도메인·테스트 기반 | Node만으로 실행, 타입·테스트·빌드 통과       |
| P2 fixture 핵심     | 네트워크 없는 대표 계획 완주       | 입력→장소→경로→검증→승인→문서 스모크 통과    |
| P3 지도형 UI        | 데스크톱 3패널·모바일 목록         | 지도 대체 UI, 선택 동기화, 접근성 검사 통과  |
| P4 에이전트         | 허용 도구·근거·상태기계            | 미등록 도구·실패·프롬프트 주입 테스트 통과   |
| P5 교육·안전·문서   | 성취기준·안전·대안·문서            | 8개 PRD 인수 시나리오 통과                   |
| P6 평가·운영        | 12개 벤치마크·평가 리포트          | JSON·Markdown 평가 결과 생성                 |
| P7 로컬 릴리스 후보 | 문서·보안·Git 정리                 | 비밀 0건, 테스트 통과, 로컬 커밋, remote 0개 |

## 승인 없이 진행하는 항목

- 작업 브랜드와 저장소 슬러그 적용
- 로컬 디렉터리·Git 저장소 초기화
- fixture 데이터와 합성 벤치마크 제작
- 도메인 모델, 규칙 엔진, 에이전트 도구 계약 구현
- 지도 대체 시각화와 조건부 Kakao SDK 연결 코드 작성
- 테스트, 보안 검사, 접근성 검사, 문서, 데모 스크립트 작성
- 로컬 빌드·스모크·평가 실행
- GitHub README, 저장소 설명·토픽·푸시 절차 준비
- 원격이 없는 로컬 커밋 생성

## 외부 승인 또는 외부 상태가 필요한 게이트

| 게이트                     | 필요한 이유                              | 그 전까지 가능한 작업                      |
| -------------------------- | ---------------------------------------- | ------------------------------------------ |
| G1 Kakao 개발자 앱·키      | 사용자 계정의 앱 활성화·도메인·과금 설정 | SDK 로더, 어댑터, fixture·계약 테스트 완성 |
| G2 공공데이터 키           | 사용자 계정 발급 또는 기관 승인          | 공급자 계약, 샘플 응답, 실패 처리 완성     |
| G3 교육과정 원문·학교 양식 | 사용·배포 범위와 공식 템플릿 확인        | 구조·샘플·사용자 입력 경로 완성            |
| G4 상표·도메인 최종 확정   | 법률·구매·선점 판단                      | 작업 브랜드로 전체 로컬 산출물 완성        |
| G5 Windows 11 실기기 검증  | 현재 작업 장비 밖의 물리적 실행          | 동일 명령과 기대 결과 문서·CI 완성         |
| G6 GitHub 원격 생성·푸시   | 외부 저장소 변경                         | 로컬 Git 커밋과 공개 전 검사 완료          |

## 중단 규칙

- 외부 계정 로그인, 키 발급, 결제수단 연결, 도메인 구매는 자동 실행하지 않는다.
- 실제 학생·학교 개인정보를 fixture나 테스트에 사용하지 않는다.
- 근거 없는 안전·접근성·가격 정보는 승인 가능 상태로 만들지 않는다.
- GitHub remote 추가, 저장소 생성, push는 사용자 승인 전 실행하지 않는다.
- 물리적으로 접근하지 못한 장비의 검증을 통과했다고 표시하지 않는다.

## 완료 정의

GitHub 업로드 직전 완료 상태는 다음과 같다.

1. fixture 모드의 대표 데모가 로컬에서 완주한다.
2. 자동화된 타입·단위·통합·브라우저·빌드·OpenSpec 검증이 통과한다.
3. 외부 API 없이도 결과의 데이터 모드와 출처가 명확하다.
4. 공개 파일에 비밀정보·학생 개인정보·장비 개인정보가 없다.
5. README, 라이선스, 운영·검증·한계 문서가 준비된다.
6. 로컬 Git 커밋이 존재하고 remote는 없다.
7. 외부 승인 게이트와 GitHub 게시 명령은 handoff에만 기록한다.
